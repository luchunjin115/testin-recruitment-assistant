# Testin云测招聘助手使用说明

Testin云测招聘助手是一个面向 HR 招聘场景的 AI 应用 Demo，主要用于解决候选人信息手动录入效率低、简历初筛耗时、招聘状态跟踪不及时、数据统计容易失真等问题。

系统核心流程是：

投递者在线填写信息
→ 进入 AI 初筛中心
→ HR 查看 AI 匹配度和风险提示
→ HR 点击通过初筛
→ 候选人进入正式候选人列表
→ HR 进行面试、复试、Offer、入职、淘汰等流程管理
→ 数据看板同步统计招聘进展

本文档面向 HR、面试官和非开发人员，重点说明这个 Demo 如何启动、如何演示、以及每个页面在真实招聘流程中的作用。

## 一、运行说明

### 1. 下载代码

可以使用以下任意一种方式获取项目代码：

第一种方式：从 GitHub 下载 ZIP 压缩包。

1. 打开项目 GitHub 页面。
2. 点击 `Code`。
3. 选择 `Download ZIP`。
4. 下载完成后解压到本地电脑。

第二种方式：使用 `git clone`。

```bash
git clone <项目仓库地址>
```

如果使用 `git clone`，请进入克隆后的项目目录继续操作。

### 2. 解压项目

如果下载的是 ZIP 文件，请先解压。解压后应能看到项目根目录中的这些文件和文件夹：

- `backend`：后端服务代码
- `frontend`：前端页面代码
- `scripts`：演示数据和工具脚本
- `start_backend.bat`：Windows 后端一键启动脚本
- `start_frontend.bat`：Windows 前端一键启动脚本
- `README.md`：项目说明

后续命令都建议在项目根目录或对应的 `backend`、`frontend` 目录中执行。

### 3. 启动后端

后端负责保存候选人、岗位、AI 初筛结果、操作日志和数据看板统计。

方式一：双击启动。

在项目根目录双击：

```bash
start_backend.bat
```

方式二：手动命令启动。

打开终端，进入项目根目录后执行：

```bash
cd backend
pip install -r requirements.txt
python run.py
```

后端启动成功后，可以访问接口文档：

```bash
http://localhost:8000/docs
```

如果能看到 FastAPI 接口文档页面，说明后端已经正常运行。

### 4. 启动前端

前端负责展示 HR 操作页面、投递者填写页面、候选人列表、AI 初筛中心和数据看板。

方式一：双击启动。

在项目根目录双击：

```bash
start_frontend.bat
```

方式二：手动命令启动。

另开一个终端，进入项目根目录后执行：

```bash
cd frontend
npm install
npm run dev
```

如果 PowerShell 拦截 `npm` 命令，可以改用：

```bash
npm.cmd install
npm.cmd run dev
```

前端启动成功后，浏览器访问：

```bash
http://localhost:5173
```

### 5. 重要说明：前端必须使用 Vite 启动

请不要使用下面这种方式启动前端：

```bash
python -m http.server
```

原因是当前前端项目依赖 Vite 开发服务器代理 API 请求。页面中的 `/api` 请求需要被转发到后端 `http://localhost:8000`。如果使用 Python `http.server`，它只能提供静态文件，不能代理 API 请求，可能导致页面打开了，但候选人、Dashboard、AI 初筛等数据加载不出来。

正确做法是使用：

```bash
npm run dev
```

或在 PowerShell 环境中使用：

```bash
npm.cmd run dev
```

### 6. 演示数据重置

如果页面打开后没有候选人数据，或者希望在演示前恢复一套干净的演示环境，可以在项目根目录执行：

```bash
python scripts/reset_demo_data.py
```

当前项目提供了这个脚本。它用于重置候选人演示数据、操作日志和简历附件，并保留或补齐岗位管理数据，方便面试演示。

## 二、页面入口说明

当前 Demo 的主要页面入口如下：

1. 投递者填写页：

```bash
http://localhost:5173/apply
```

用于模拟候选人在线投递。候选人在这里填写基本信息、教育背景、应聘岗位、技能关键词、自我介绍，并可选择上传简历。

2. 数据看板：

```bash
http://localhost:5173/dashboard
```

用于 HR 查看整体招聘进展，包括总投递、待初筛、正式流程人数、招聘漏斗、渠道分布、最近新增、超时未跟进和最近操作日志。

3. 岗位管理：

```bash
http://localhost:5173/jobs
```

用于 HR 统一维护岗位库。投递者填写页中的岗位下拉框来自这里的启用岗位。

4. AI 初筛中心：

```bash
http://localhost:5173/ai-screening
```

用于 HR 处理新投递和待初筛候选人，查看 AI 匹配度、推荐等级、初筛建议和风险提示。

5. 候选人列表：

```bash
http://localhost:5173/candidates
```

用于 HR 管理已经通过初筛并进入正式招聘流程的候选人，也可以查看备选候选人和淘汰候选人。

6. 候选人详情：

候选人详情没有固定单一入口。可以在候选人列表或 AI 初筛中心点击候选人姓名进入，也可以点击详情按钮进入。

示例地址形式：

```bash
http://localhost:5173/candidates/1
```

其中 `1` 是候选人的系统 ID。

## 三、完整业务使用流程

### 流程 1：投递者在线填写信息

投递者填写页是整个招聘流程的起点。

1. 打开投递者填写页：

```bash
http://localhost:5173/apply
```

2. 填写基本信息。

页面会要求或允许候选人填写：

- 姓名：必填
- 手机号：必填，页面会校验手机号格式
- 邮箱：选填，页面会校验邮箱格式

3. 填写教育背景。

候选人可以填写：

- 学校
- 学历
- 专业

4. 填写求职信息。

候选人需要选择或填写：

- 应聘岗位：必填，来自岗位管理中当前启用的岗位
- 技能关键词：可输入多个技能，例如 `Python`、`React`、`接口测试`
- 自我介绍：可填写个人优势、项目经历和求职说明

5. 上传简历。

当前项目支持上传：

- PDF
- DOCX
- TXT

简历上传是可选项，不上传也可以提交。当前页面限制单个简历文件最大 10MB。

如果候选人上传了简历，系统会把简历附件和候选人记录关联起来，后续 HR 可以在候选人详情页查看或下载。

6. 点击“提交投递”。

提交成功后，页面会显示“投递成功”的提示。

7. 系统自动创建候选人记录。

候选人提交后，系统会在后端创建一条候选人记录，记录来源通常为“在线投递”，并保存投递时间、应聘岗位、技能关键词、自我介绍和简历附件信息。

8. 新投递候选人默认进入 AI 初筛中心。

新投递候选人不会直接进入正式候选人列表，而是先进入 AI 初筛中心，等待 HR 初筛确认。

这样设计的原因是：真实招聘中，新投递候选人数量可能很多，如果全部直接进入正式候选人列表，会增加 HR 管理压力，也会让正式流程列表变得混乱。因此系统先让新投递候选人进入 AI 初筛中心，由 AI 辅助判断岗位匹配度、风险点和推荐等级，HR 确认通过后再进入正式流程。

## 四、岗位管理使用说明

岗位管理用于 HR 统一维护招聘岗位，避免候选人在投递时自由输入岗位名称造成统计混乱。

### 1. 打开岗位管理页面

访问：

```bash
http://localhost:5173/jobs
```

页面会展示当前岗位列表。

### 2. 查看当前岗位

岗位列表中可以看到：

- 岗位名称
- 所属部门
- 岗位描述
- 岗位要求
- 筛选标准
- 岗位状态
- 关联候选人数
- 更新时间

岗位状态分为：

- 启用：投递者页面可以选择该岗位
- 停用：投递者页面不能再选择该岗位

### 3. 新增岗位

点击“新增岗位”，可以维护岗位信息。

当前项目支持填写：

- 岗位名称
- 所属部门
- 岗位描述
- 岗位要求
- 必备技能
- 加分技能
- 学历要求
- 经验要求
- 岗位关键词
- 风险提示关键词
- 岗位状态

这些信息不仅用于岗位展示，也会参与 AI 初筛判断。例如，AI 会参考必备技能、加分技能、学历要求、经验要求和风险提示关键词，生成匹配度、推荐等级和风险提示。

### 4. 修改岗位

在岗位列表中点击“编辑”，可以修改岗位名称、部门、描述、要求、筛选标准和状态。

修改岗位后，投递者页面刷新后会读取最新岗位信息。

### 5. 启用 / 停用岗位

在岗位列表中可以点击“启用”或“停用”。

启用岗位后：

- 该岗位会出现在 `/apply` 页面的岗位下拉框中
- 候选人可以选择该岗位投递

停用岗位后：

- 该岗位不会再出现在 `/apply` 页面的岗位下拉框中
- 新候选人不能再选择该岗位
- 历史候选人的岗位数据不会丢失

如果岗位已经有关联候选人，删除操作会倾向于停用岗位，以保留历史数据，避免破坏已有候选人的投递记录。

### 6. 岗位管理和投递者页面的关系

岗位管理与投递者填写页是联动的：

- 只有启用的岗位会出现在 `/apply` 页面的岗位下拉框中
- HR 修改岗位后，投递者页面刷新后会看到最新岗位
- 停用岗位后，投递者不能再选择该岗位
- 历史候选人的岗位名称和投递记录不会丢失

岗位统一管理可以避免候选人自由输入岗位名称导致数据混乱。例如，如果没有统一岗位库，“测试工程师”“软件测试”“QA”可能被统计成三个不同岗位，导致数据看板中的岗位分布失真。统一岗位管理可以让投递、初筛、候选人列表和 Dashboard 使用更一致的数据口径。

## 五、AI 初筛中心使用说明

AI 初筛中心用于处理：

- 新投递候选人
- 待初筛候选人
- 尚未进入正式招聘流程的候选人

AI 初筛中心不用于处理正式流程中的备选候选人。备选候选人属于 HR 已经完成初筛后，在正式流程中暂缓推进的候选人，应在候选人列表中管理。

访问地址：

```bash
http://localhost:5173/ai-screening
```

页面顶部会提示：AI 初筛中心只处理新投递和待初筛候选人。AI 只提供匹配度和风险提示，最终通过或初筛淘汰由 HR 确认。

### 1. 招聘漏斗 / 初筛统计

页面上方有“招聘漏斗初筛统计”，用于展示整体初筛情况。

当前项目展示的统计包括：

- 总投递人数：系统中有效候选人的总数
- 待初筛：还在等待 HR 初筛处理的人数
- 已通过初筛：已经通过初筛并进入后续流程的人数
- 初筛淘汰：在初筛阶段被淘汰的人数
- AI 已评分：已经生成 AI 初筛结果的人数

这些统计帮助 HR 快速判断当前招聘漏斗是否堵在初筛环节。

### 2. 当前处理池

“当前处理池”说明当前列表中有多少人正在等待 HR 初筛处理，以及其中多少人已经由 AI 评分、多少人尚未评分。

它还会展示：

- 高优先级人数
- 中优先级人数
- 低优先级人数
- 尚未评分人数

如果 HR 设置了筛选条件，当前处理池会反映筛选后的结果。

### 3. 筛选和排序条件

当前项目的 AI 初筛中心支持以下筛选和排序：

- 按岗位搜索
- 按投递时间筛选：全部时间、今日投递、本周投递、最近 7 天、最近 30 天
- 按推荐等级筛选：高优先级、中优先级、低优先级
- 按处理状态筛选：待初筛、初筛淘汰
- 按初筛状态筛选：已初筛、尚未初筛
- 按排序方式查看：最新投递优先、最早投递优先、匹配度从高到低、匹配度从低到高、最近初筛优先

当前页面没有单独的候选人关键词搜索框。如果需要按姓名、学校、专业、技能、邮箱或手机号搜索，应使用候选人列表页面。

### 4. 候选人表格字段

AI 初筛中心表格用于让 HR 快速比较待处理候选人。主要字段含义如下：

- 姓名：候选人姓名，点击后进入候选人详情页
- 应聘岗位：候选人投递时选择的岗位
- 匹配依据：AI 根据岗位标准生成的匹配说明，并可查看岗位标准
- 投递时间：候选人进入系统的时间
- 学校：候选人的学校信息
- 技能标签：候选人填写或简历解析出的技能关键词
- AI 匹配度：AI 生成的 0-100 分匹配度
- 推荐等级：高优先级、中优先级、低优先级或未初筛
- 初筛建议：AI 给出的初筛建议，例如建议初筛、暂缓或待 HR 复核
- 处理状态：待初筛、已通过、初筛淘汰等
- 风险提示：AI 提醒 HR 关注的风险点
- AI 理由：AI 对评分和建议的解释
- 操作：通过初筛、初筛淘汰、详情、重新初筛

### 5. AI 匹配度

AI 匹配度是系统根据候选人信息、技能关键词、简历内容和岗位要求生成的 0-100 分。

匹配度越高，表示候选人与岗位要求越接近。匹配度不是录用结论，只是帮助 HR 快速排序和定位重点候选人。

### 6. 推荐等级

推荐等级是 AI 对候选人优先级的辅助判断。

- 高优先级：建议 HR 优先查看
- 中优先级：候选人有一定匹配度，可以进一步判断
- 低优先级：可暂缓处理，或在候选人较少时再查看
- 未初筛：尚未生成 AI 初筛结果

### 7. 初筛建议

初筛建议用于帮助 HR 快速理解 AI 的判断方向。

常见含义包括：

- 建议初筛：候选人与岗位要求较匹配，可考虑进入后续流程
- 暂缓：匹配度不够明确，建议 HR 结合简历再判断
- 待 HR 复核：AI 不能直接做决定，需要 HR 查看详细信息

实际页面会根据 AI 初筛结果展示对应标签。

### 8. 操作按钮说明

#### 通过初筛

点击“通过初筛”后，系统会执行以下动作：

- 候选人的 `screening_status` 变为 `passed`
- 招聘阶段变为“待约面”
- 候选人从 AI 初筛中心的默认待处理池中移出
- 候选人进入候选人列表的正式流程候选人范围
- 系统写入操作日志和阶段变更记录
- 数据看板同步更新

这个动作表示 HR 认可该候选人可以进入正式招聘流程。

#### 初筛淘汰

点击“初筛淘汰”后，系统会弹出确认窗口，要求填写淘汰原因。

确认后：

- 候选人不会进入正式候选人列表的正式流程范围
- 候选人数据不会删除
- 后续可在淘汰候选人范围或相关记录中查看
- 系统写入操作日志和阶段变更记录
- 数据看板同步更新

淘汰原因必须填写，这样后续复盘时可以追溯为什么候选人没有继续推进。

#### 重新初筛

点击“重新初筛”后，系统会重新生成：

- AI 匹配度
- 推荐等级
- 初筛建议
- 风险提示
- AI 理由

重新初筛不会直接改变候选人是否通过初筛，也不会自动淘汰候选人。最终仍由 HR 点击“通过初筛”或“初筛淘汰”。

#### 详情

点击“详情”可以进入候选人详情页，查看候选人的完整资料、简历附件、AI 摘要、AI 标签、AI 跟进建议、操作日志和阶段变更记录。

### 9. 批量初筛

页面右上方有“开始批量初筛”按钮。

点击后，系统会对当前筛选结果中的候选人执行 AI 初筛。已经初筛过的候选人也会重新生成结果。批量初筛适合演示或大量候选人进入系统后的集中处理。

### 10. AI 只做辅助，不自动决定

AI 初筛结果仅作为 HR 辅助参考，不会自动决定候选人是否通过或淘汰。最终操作仍由 HR 确认。

这点非常重要：系统强调的是“AI 辅助 HR 提效”，不是让 AI 替代 HR 做招聘决策。

## 六、候选人列表使用说明

候选人列表用于管理已经通过初筛、进入正式招聘流程的候选人。

访问地址：

```bash
http://localhost:5173/candidates
```

候选人列表和 AI 初筛中心的区别是：

- AI 初筛中心处理新投递、待初筛、尚未进入正式流程的人
- 候选人列表管理已经通过初筛、进入正式招聘流程的人，以及备选和淘汰记录

当前候选人列表保留以下范围：

1. 正式流程候选人
2. 备选候选人
3. 淘汰候选人

### 1. 正式流程候选人

正式流程候选人指通过初筛后进入招聘流程的候选人，例如：

- 待约面
- 已约面
- 面试中
- 复试
- offer
- 入职

这些候选人已经不属于“待初筛池”，HR 需要在这里继续推进面试、复试、Offer、入职或淘汰。

### 2. 备选候选人

备选候选人指已经通过初筛并进入正式流程后，HR 暂时不推进但保留观察的候选人。

注意：备选不是 AI 初筛中心阶段。只有进入正式流程后，才考虑是否标记为备选。

### 3. 淘汰候选人

淘汰候选人指已明确不继续推进，但系统保留记录以便追溯。

真实业务中，不建议直接删除候选人。更推荐使用淘汰或归档方式保留记录，这样可以保证数据可追溯、统计口径完整，也方便后续复盘。

当前页面确实提供删除按钮，但演示和真实业务说明时建议强调：删除是强操作，应谨慎使用；常规招聘流程优先使用淘汰。

### 4. 搜索候选人

候选人列表支持关键词搜索。当前项目的关键词搜索覆盖：

- 姓名
- 学校
- 专业
- 技能
- 邮箱
- 手机号
- 应聘岗位

例如，HR 可以输入某个技能关键词，快速找到相关候选人。

### 5. 岗位筛选

可以按应聘岗位筛选候选人。

岗位选项来自系统中的岗位和候选人历史数据，方便 HR 查看某一个岗位下的招聘进展。

### 6. 阶段筛选

在“正式流程候选人”范围内，可以按正式流程阶段筛选。

当前正式流程阶段包括：

- 待约面
- 已约面
- 面试中
- 复试
- offer
- 入职

候选人列表的正式流程筛选不应出现：

- 新投递
- 待初筛
- 待筛选

这些阶段应在 AI 初筛中心或候选人详情中的初筛流程中处理。

### 7. 来源筛选

可以按来源渠道筛选候选人。

当前项目支持的来源包括在线投递、HR 录入、Boss 直聘、内推等，实际选项会根据系统已有候选人数据动态展示。

### 8. 新增筛选

候选人列表支持“今日新增”筛选。

列表中还会展示新增标签：

- 今日新增：当天新增候选人
- 新投递：最近 3 天新增但不是今天新增的候选人

### 9. 批量操作

候选人列表支持选择多名候选人后批量处理。

当前项目实际支持的批量操作包括：

- 批量标记备选：仅正式流程候选人可使用
- 批量标记待约面：将选中候选人统一标记为待约面
- 批量淘汰：批量淘汰候选人，可填写淘汰原因
- 批量导出 CSV：导出选中候选人
- 批量生成 AI 跟进建议：为正式流程候选人批量生成跟进建议

当前候选人列表页面没有“批量通过初筛”按钮。通过初筛应在 AI 初筛中心处理。

### 10. 同步和导出

候选人列表右上方提供：

- 同步 CSV
- 导出

这些功能用于演示数据同步和候选人数据导出。

## 七、候选人详情页使用说明

候选人详情页用于查看单个候选人的完整信息、AI 分析结果、简历附件、HR 操作入口、阶段变更记录和操作日志。

进入方式：

- 在 AI 初筛中心点击候选人姓名或“详情”
- 在候选人列表点击候选人姓名
- 在候选人列表双击候选人所在行

### 1. 基本信息

详情页展示候选人的基础资料，包括：

- 姓名
- 手机号
- 邮箱
- 学校
- 学历
- 专业
- 应聘岗位
- 工作年限
- 来源渠道
- 录入时间
- 最近更新时间
- 技能
- 工作经历
- 自我介绍
- HR 备注

如果系统识别到重复候选人，也会在页面上方提示重复记录。

### 2. 投递与 AI 初筛信息

详情页会展示候选人的投递和初筛状态，包括：

- 投递来源
- 当前阶段
- 初筛处理状态
- 匹配度
- 推荐等级
- 初筛建议
- AI 理由
- 风险提示
- 最近动作时间

如果候选人还处于 AI 初筛阶段，页面会提示“当前处于 AI 初筛阶段”，并说明候选人尚未进入正式招聘流程。

### 3. 简历附件

如果候选人上传了简历，可以在详情页看到简历附件信息：

- 文件名
- 文件类型
- 文件大小
- 上传时间
- 查看说明

当前项目支持 PDF 和 TXT 直接查看；DOCX 等格式通常建议下载后查看。

如果候选人没有上传简历，页面会显示“未上传简历”。

### 4. AI 摘要

AI 摘要用于帮助 HR 快速了解候选人。

系统会根据候选人的基础信息、技能、经历和岗位信息生成简要总结。HR 可以点击“重新生成”刷新摘要。

AI 摘要适合在候选人较多时快速浏览，但不替代 HR 对完整简历和面试反馈的判断。

### 5. AI 标签

AI 标签用于给候选人打上结构化标签，例如学历、经验、技能、岗位方向或特殊标记等。

HR 可以点击“重新生成”刷新标签。标签可以帮助 HR 快速识别候选人特点。

### 6. AI 跟进建议

AI 跟进建议会根据候选人的当前阶段、更新时间、是否缺少信息、是否超时未跟进等因素生成下一步建议。

例如：

- 新投递候选人建议尽快初筛
- 待约面候选人建议尽快联系安排面试
- Offer 阶段候选人建议跟进接受意向
- 超时未跟进候选人建议优先处理

详情页会展示跟进优先级、超时标签、超时天数、触发原因和建议内容。HR 可以点击“刷新建议”重新生成。

### 7. HR 决策操作

详情页右侧提供 HR 决策操作。不同阶段可用按钮不同，不适合当前阶段的按钮会被禁用。

当前项目支持：

- 通过初筛
- 安排面试
- 进入复试
- 发放 Offer
- 确认入职
- 淘汰

每个操作提交后，系统会自动：

- 更新候选人阶段
- 保存备注、反馈或业务字段
- 写入阶段变更记录
- 写入操作日志
- 同步数据看板统计

### 8. 通过初筛

适用于新投递或待筛选阶段的候选人。

点击后，候选人会进入“待约面”阶段，并进入正式候选人列表。

### 9. 安排面试

适用于“待约面”阶段。

需要或可以填写：

- 面试时间：必填
- 面试方式：线上面试、线下面试或电话面试
- 面试官
- 面试备注

提交后，候选人阶段会推进到“已约面”。

### 10. 进入复试

适用于“已约面”或“面试中”阶段。

可以填写：

- 复试原因 / 一面反馈
- 复试时间
- 复试面试官

提交后，候选人阶段会推进到“复试”。

### 11. 发放 Offer

适用于“已约面”“面试中”或“复试”阶段。

需要或可以填写：

- Offer 岗位：必填，默认可使用候选人应聘岗位
- 薪资范围
- 预计入职时间
- Offer 备注

提交后，候选人阶段会推进到 `offer`。

### 12. 确认入职

适用于 `offer` 阶段。

可以填写：

- 实际入职时间
- 入职备注

提交后，候选人阶段会推进到“入职”，招聘流程结束。

### 13. 淘汰

适用于未结束的招聘阶段。

淘汰时必须填写淘汰原因，也可以填写备注。

提交后：

- 候选人阶段变为“淘汰”
- 流程结束
- 系统保留候选人记录
- 系统写入阶段变更记录和操作日志
- Dashboard 同步更新统计

### 14. 阶段人工修正

详情页提供“阶段人工修正”功能。

它用于修正系统自动判断有误的情况。演示时可以说明：常规业务应优先使用 HR 决策操作，人工修正适合特殊数据修复场景。

### 15. 阶段变更记录

每次状态变化都会记录：

- 原阶段
- 新阶段
- 变更原因
- 触发来源，例如 HR 操作、系统自动或人工修正
- 变更时间

这可以帮助 HR 或管理者追踪候选人的完整流转过程。

### 16. 操作日志

操作日志记录 HR 操作和系统行为，例如：

- 创建候选人
- 通过初筛
- 安排面试
- 进入复试
- 发放 Offer
- 生成 AI 面试总结
- 淘汰候选人

操作日志保证招聘过程可追溯，减少“谁在什么时候做了什么操作”无法确认的问题。

### 17. 面试反馈 AI 总结

详情页还提供“面试反馈 AI 总结”区域。

HR 可以填写：

- 面试官
- 面试轮次
- 反馈时间
- 面试原始反馈

点击“生成 AI 面试总结”后，系统会生成：

- 技术能力总结
- 沟通表达总结
- 岗位匹配度判断
- 风险点
- 推荐结论
- 下一步建议

AI 面试总结仅作为辅助参考，不会自动改变候选人阶段，也不会自动淘汰候选人。

## 十、常见问题

### 1. 页面打开但没有候选人数据怎么办？

请按顺序检查：

1. 后端是否已经启动。
2. 是否能打开 `http://localhost:8000/docs`。
3. 前端是否使用 `npm run dev` 或 `start_frontend.bat` 启动。
4. 是否误用了 `python -m http.server`。
5. 如果数据库为空，可以运行演示数据重置脚本：

```bash
python scripts/reset_demo_data.py
```

### 2. 前端为什么不能用 python http.server？

因为 Python `http.server` 只能提供静态文件，不能把 `/api` 请求代理到后端。

当前前端页面需要通过 Vite 把 `/api` 请求转发到：

```bash
http://localhost:8000
```

如果用 `python -m http.server`，页面可能能打开，但候选人列表、Dashboard、AI 初筛中心等数据会加载失败。

### 3. npm 命令被 PowerShell 拦截怎么办？

可以使用：

```bash
npm.cmd install
npm.cmd run dev
```

也可以双击项目根目录的：

```bash
start_frontend.bat
```

### 4. 后端接口打不开怎么办？

请确认后端是否启动。

可以在项目根目录双击：

```bash
start_backend.bat
```

或手动执行：

```bash
cd backend
pip install -r requirements.txt
python run.py
```

然后访问：

```bash
http://localhost:8000/docs
```

如果该地址打不开，说明后端没有成功运行，前端页面也无法正常获取数据。

### 5. 如何重置演示数据？

当前项目支持重置演示数据。

在项目根目录执行：

```bash
python scripts/reset_demo_data.py
```

重置后会恢复一套适合演示的候选人、简历附件、初筛状态和招聘流程数据。

### 6. AI 功能是否需要真实 API Key？

当前 Demo 默认可使用 mock 模式演示，不需要真实 API Key。

如果需要接入真实模型，可以复制 `.env.example` 为 `.env`，并在 `.env` 中配置对应模型服务的 API Key 和模型参数。

### 7. 为什么新投递候选人不直接进入候选人列表？

为了符合真实招聘漏斗。

新投递候选人数量通常较多，如果全部直接进入候选人列表，会让正式流程变得混乱。当前系统让新投递先进入 AI 初筛中心，由 AI 辅助生成匹配度、推荐等级和风险提示，HR 点击“通过初筛”后，再进入候选人列表正式流程。

### 8. 为什么备选不在 AI 初筛中心？

备选属于进入正式招聘流程后的 HR 判断，不属于初筛前处理。

AI 初筛中心处理的是新投递和待初筛候选人；备选候选人说明 HR 已经完成初筛并决定暂缓推进，应在候选人列表中管理。

### 9. 候选人没有上传简历还能投递吗？

可以。

当前投递页的简历上传是可选项。没有上传简历时，系统仍会创建候选人记录，但详情页会显示“未上传简历”。HR 后续可以结合候选人填写的信息进行初筛。

### 10. AI 初筛会自动淘汰候选人吗？

不会。

AI 只生成匹配度、推荐等级、初筛建议和风险提示。是否通过初筛或淘汰，必须由 HR 手动点击确认。

## 十一、项目价值总结

Testin云测招聘助手的价值主要体现在：

1. 减少 HR 手动录入：投递者可以在线填写信息，系统自动创建候选人记录。
2. 用 AI 初筛减少逐份查看简历的时间：AI 提供匹配度、推荐等级、初筛建议和风险提示。
3. 岗位统一管理，减少岗位名称混乱造成的数据失真。
4. 新投递先进入 AI 初筛中心，正式流程列表更清晰。
5. HR 操作自动写日志，招聘过程可追溯。
6. 数据看板同步统计，减少人工维护表格。
7. AI 跟进建议和超时提醒，减少遗漏候选人。
