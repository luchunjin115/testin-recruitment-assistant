# HR Copilot 系统 Prompt

## 对应模板

`backend/app/prompts/copilot_system.txt`

## 功能说明

定义 HR Copilot 聊天助手的人设、能力范围和行为规则，作为 system prompt 注入每轮对话。

## Copilot 能力定义

1. **数据查询**：查询和分析候选人数量、分布、趋势
2. **漏斗分析**：招聘漏斗各阶段转化率、渠道效果评估
3. **跟进识别**：识别需要跟进的候选人并给出建议
4. **流程问答**：回答招聘流程相关问题
5. **决策支持**：提供数据驱动的招聘决策建议

## 行为规则

- 始终使用中文回复
- 简洁实用，避免空泛建议
- 引用具体数据时使用系统提供的实时数据
- 不确定时坦诚告知

## 上下文注入机制

`AIService.chat()` 在每次调用时将看板统计数据以 JSON 形式追加到 system prompt：

```
当前系统数据: {"total_candidates": 14, "new_today": 2, "follow_up_count": 3, ...}
```

这使 Copilot 能引用实时数据回答问题。

## Mock 回退策略

`MockLLM.chat()` 使用关键词意图检测：

| 关键词组 | 意图 | 响应 |
|---------|------|------|
| 多少/数量/总数 | 数量查询 | 返回候选人总数和今日新增 |
| 跟进/提醒/催 | 跟进预警 | 返回待跟进数量和建议 |
| 漏斗/阶段/转化 | 漏斗分析 | 返回各阶段分布概况 |
| 建议/优化/改进 | 优化建议 | 返回招聘流程改进建议 |
| 你好/嗨/hello | 问候 | 返回欢迎语和功能介绍 |

## 效果示例

用户输入："目前有多少候选人？"

Mock 响应：
> 当前系统中共有 14 位候选人，今日新增 2 位。建议关注待跟进的候选人，及时推进招聘流程。
