# 自动标签生成 Prompt

## 对应模板

`backend/app/prompts/auto_tagging.txt`

## 功能说明

根据候选人信息生成 3-6 个分类标签，用于快速筛选和人才分类。

## 标签分类体系

| 类别 | 示例标签 |
|------|---------|
| 学历标签 | 985高校、211高校、硕士学历、博士学历、海外背景 |
| 经验标签 | 应届/实习、1-3年经验、3年+经验、资深经验、管理经验 |
| 技能标签 | Python、Java、前端开发、自动化测试、性能测试、AI方向、全栈能力 |
| 岗位标签 | 测试方向、开发方向、产品方向、数据分析 |
| 特殊标签 | 高匹配度、急需跟进、潜力候选人 |

## 设计要点

1. **分类引导**：列出 5 大类标签参考，引导 LLM 从多维度打标
2. **数量控制**：3-6 个标签，避免过多（信息噪声）或过少（缺乏区分度）
3. **输出格式**：JSON 数组 `["标签1", "标签2"]`，便于前端直接渲染
4. **字数限制**：每个标签不超过 6 个字，确保 UI 展示整洁

## Mock 回退策略

`MockLLM.generate_tags()` 使用规则引擎：

1. **学历标签**：匹配 985 名校名单（清华、北大等 10 所），或检测学历等级
2. **经验标签**：按 experience_years 分级（0→应届、1-3→初级、3+→资深）
3. **技能标签**：关键词映射表（15 组映射，如 Python→Python、test/测试→自动化测试）
4. **岗位标签**：根据 target_role 关键词推断方向
5. **上限控制**：最终截取前 6 个标签

## 效果示例

输入：3 年经验、北京邮电大学、Python/Selenium 技能栈

输出：`["211高校", "3年+经验", "Python", "自动化测试", "测试方向"]`
