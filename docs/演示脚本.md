# 演示脚本

> 总时长约 5 分钟，分 5 个步骤演示系统核心功能。

---

## 准备工作

### 启动方式 A：Docker（推荐）

```bash
docker-compose up --build
# 前端：http://localhost:3000
# 后端 Swagger：http://localhost:8000/docs
```

### 启动方式 B：手动启动

```bash
# 终端 1 - 后端
cd backend
pip install -r requirements.txt
python run.py
# → http://localhost:8000/docs

# 终端 2 - 前端
cd frontend
npm install
npm run dev
# → http://localhost:5173
```

系统启动后会自动初始化数据库并导入 14 条种子候选人数据。

---

## 第一步：招聘看板总览（约 1 分钟）

**操作**：打开首页（Dashboard）

**展示要点**：
1. **统计卡片**：候选人总数、今日新增、待跟进数、本周入职数
2. **招聘漏斗图**：展示从"新投递"到"入职"各阶段的候选人分布
3. **渠道饼图**：展示各招聘渠道（Boss直聘、拉勾、内推等）的候选人占比
4. **跟进预警**：高亮显示超过 3 天未跟进的候选人
5. **今日摘要**：AI 生成的今日招聘概况
6. **最近操作日志**：系统操作记录流水

**演示话术**：
> "这是招聘看板，HR 打开系统就能一目了然地看到整体招聘情况。统计卡片展示关键指标，漏斗图展示各阶段转化情况，预警模块自动提醒需要跟进的候选人。"

---

## 第二步：表单录入候选人（约 1 分钟）

**操作**：点击侧边栏"录入候选人"

**展示要点**：
1. 填写基本信息：姓名、手机号、邮箱、学校、学历等
2. 选择来源渠道和招聘阶段
3. 提交后系统自动进行去重检查（三级策略：手机号 > 邮箱 > 姓名+学校+岗位）
4. 自动生成 AI 摘要和标签

**演示话术**：
> "HR 也可以手动录入候选人。系统自动进行三级去重检查，避免重复录入。提交后自动触发 AI 处理。"

---

## 第三步：简历上传与 AI 解析（约 1 分钟）

**操作**：点击侧边栏"简历上传"

**展示要点**：
1. 上传示例简历文件（`sample_data/sample_resume_1.txt`）
2. 系统自动调用 AI 解析简历，提取结构化信息
3. 展示解析预览：姓名、手机号、学校、技能等字段
4. 确认无误后点击"确认入库"

**演示话术**：
> "这是简历解析功能。上传简历后，AI 自动提取关键信息并结构化展示。HR 确认后一键入库，无需手动录入。整个过程使用 Mock 模式，不需要任何 API Key。"

---

## 第四步：候选人列表管理（约 1 分钟）

**操作**：点击侧边栏"候选人列表"

**展示要点**：
1. **筛选功能**：按阶段（新投递/初筛/面试中等）、渠道、关键词筛选
2. **排序功能**：按创建时间、更新时间排序
3. **内联编辑**：直接在列表中修改候选人的招聘阶段
4. **分页导航**：支持大量数据浏览
5. 点击某个候选人进入详情页

**演示话术**：
> "候选人列表支持多维度筛选和排序，HR 可以快速定位目标候选人。阶段修改可以直接在列表内联完成，操作高效。"

---

## 第五步：AI 功能与 Copilot 对话（约 1 分钟）

**操作**：进入某个候选人的详情页

**展示要点**：
1. **AI 摘要**：点击"生成摘要"按钮，展示 AI 生成的候选人概况
2. **AI 标签**：点击"生成标签"按钮，展示自动分类标签
3. **跟进建议**：展示 AI 基于当前阶段给出的下一步建议
4. **操作日志**：展示该候选人的完整操作历史

**Copilot 对话演示**（在页面底部或侧边）：
- 输入："目前有多少候选人？" → 返回总数和今日新增
- 输入："有哪些需要跟进的？" → 返回待跟进候选人
- 输入："有什么优化建议？" → 返回招聘流程改进建议

**演示话术**：
> "每个候选人都可以一键生成 AI 摘要和标签。HR Copilot 支持自然语言对话，可以直接问数据、问建议，就像有一个 AI 招聘助理随时在线。"
