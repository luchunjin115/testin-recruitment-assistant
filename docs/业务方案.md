# 业务方案

## 一、问题分析

### 核心痛点

当前招聘流程中，HR 面临以下问题：

1. **全人工流程效率低**：每日手动在企业微信群、腾讯在线文档中录入候选人数据，过程繁琐耗时
2. **数据失真**：人工录入易出错，导致候选人信息不准确，影响招聘决策
3. **跟进遗漏**：候选人数量多时，容易忘记跟进某些候选人，错失优秀人才
4. **缺乏数据洞察**：无法快速了解招聘漏斗、渠道效果等关键指标

### 业务场景

- 工作载体：企业微信、企业微信群、腾讯在线文档
- 数据来源：Boss直聘、拉勾、猎聘、内推、表单录入、简历上传
- 管理对象：候选人全生命周期（新投递 → 初筛 → 面试 → offer → 入职/淘汰）

---

## 二、解决方案

### 设计理念

在**最小成本预算**下，构建一套 AI 驱动的招聘效能助手，实现：

- **零 API 成本启动**：内置 Mock LLM 模式，无需任何 API Key 即可运行完整功能
- **渐进式升级**：支持一键切换真实 LLM（OpenAI / DeepSeek / 智谱），按需付费
- **Docker 一键部署**：`docker-compose up --build` 即可完成部署

### 系统架构

```
用户(HR) → React 前端 → FastAPI 后端 → AI 服务层 → 数据库
                                          ↓
                                    Mock LLM / 真实 LLM
```

---

## 三、五大 AI 功能

### 1. 智能简历解析

- 支持 PDF / DOCX / TXT 格式简历上传
- AI 自动提取 11 个标准化字段（姓名、手机、学校、技能等）
- 解析结果可预览确认后再入库

### 2. AI 候选人摘要

- 一键生成 2-3 句专业摘要
- 突出教育背景、核心技能和岗位匹配度
- 帮助 HR 快速了解候选人概况

### 3. 自动标签分类

- 自动生成 3-6 个多维度标签
- 涵盖学历、经验、技能、岗位、特殊标记 5 大类
- 支持按标签快速筛选候选人

### 4. 智能跟进建议

- 根据候选人当前阶段生成具体的下一步建议
- 自动识别超过 3 天未跟进的候选人并预警
- 个性化建议结合候选人背景

### 5. HR Copilot 对话

- 自然语言查询招聘数据（"目前有多少候选人？"）
- 招聘漏斗分析和渠道效果评估
- 数据驱动的招聘决策建议

## 四、数据同步方案

### Adapter 模式设计

```
SyncAdapter (抽象接口)
├── LocalSheetAdapter  → 本地 CSV 文件（已实现）
└── TencentDocsAdapter → 腾讯在线文档（预留接口）
```

- **当前实现**：`LocalSheetAdapter` 将候选人数据导出为 CSV 文件，支持下载
- **扩展路径**：实现 `TencentDocsAdapter`，通过腾讯文档 API 实现双向同步，对接企业微信生态

## 五、成本分析

| 项目 | Mock 模式 | 真实 LLM 模式 |
|------|----------|-------------|
| LLM API 费用 | 0 元 | 按调用量付费（约 0.01-0.1 元/次） |
| 服务器 | 本地 / Docker | 本地 / Docker / 云服务器 |
| 数据库 | SQLite（零运维） | SQLite / PostgreSQL |
| 总计 | **完全免费** | 极低成本 |

### 关键优势

- Mock 模式已内置逼真的模板+正则引擎，**无需任何外部依赖**即可完成完整演示
- 真实 LLM 模式兼容 OpenAI API 协议，支持国产大模型（DeepSeek、智谱）
- 任何一步 LLM 调用失败都会自动回退到 Mock 模式，确保系统可用性
