## 第一部分：痛点分析（1 分钟）

各位评委好，我的方案是"Testin云测招聘助手"。

我们来看当前招聘流程的核心痛点：

**第一，全人工流程效率低。** HR 每天需要手动在企业微信群和腾讯在线文档中录入候选人数据，一个候选人就要花 5-10 分钟记录，遇到高峰期非常耗时。

**第二，数据质量差。** 人工录入容易出错——手机号抄错、学历填错、候选人重复录入，这些问题直接影响后续的招聘决策。

**第三，跟进容易遗漏。** 当候选人数量多了之后，HR 很容易忘记跟进某些候选人，导致优质人才流失。

这些问题的本质是：**HR 的时间被消耗在低价值的数据录入上，而不是高价值的招聘决策上。**

---

## 第二部分：方案介绍（1 分钟）

针对这些痛点，我设计了一套 AI 驱动的解决方案，核心是 **5 个 AI 功能**：

1. **智能简历解析**：上传简历，AI 自动提取姓名、手机、学校、技能等 11 个字段，替代手动录入
2. **AI 候选人摘要**：一键生成专业摘要，HR 不用逐个翻阅简历
3. **自动标签分类**：从学历、经验、技能等维度自动打标，方便快速筛选
4. **智能跟进建议**：根据候选人当前阶段自动生成下一步行动建议，超 3 天未跟进自动预警
5. **HR Copilot**：自然语言对话查询招聘数据，比如直接问"有多少候选人需要跟进"

**最重要的一点**：整个系统内置了 Mock LLM 模式，**不需要任何 API Key，零成本即可运行全部功能**。需要更强的 AI 能力时，只需修改一个环境变量就能切换到真实大模型。

## 第四部分：技术亮点（30 秒）

1. **双模 AI 架构**：Mock 模式零成本演示，真实 LLM 模式一键切换，任何 LLM 调用失败自动回退 Mock
2. **Prompt 工程**：5 套精心设计的 Prompt 模板，结构化输出、角色约束、格式强制
3. **三级去重**：手机号 → 邮箱 → 姓名+学校+岗位，逐级降低误判率
4. **Adapter 模式**：数据同步层预留了腾讯文档 API 接口，可无缝扩展到企业微信生态
5. **Docker 一键部署**：`docker-compose up` 即可启动完整应用

## Q&A 预案

### Q：Mock 模式和真实 LLM 有什么区别？
A：Mock 模式使用正则表达式和模板引擎，在结构化场景下准确率很高。真实 LLM 在复杂简历和自然语言对话场景下表现更好。切换只需修改 `LLM_PROVIDER` 环境变量。

### Q：为什么选择 SQLite？
A：面试项目追求零运维和快速部署，SQLite 不需要额外安装数据库服务。生产环境可以无缝迁移到 PostgreSQL，只需修改 `DATABASE_URL`。

### Q：腾讯文档集成怎么做？
A：系统已预留 `TencentDocsAdapter` 接口，实现腾讯文档 API 的读写方法即可。Adapter 模式确保核心逻辑不受影响。

### Q：简历解析准确率如何？
A：Mock 模式下，对格式规范的简历（有明确的姓名、手机号、学校等信息），关键字段提取准确率在 90% 以上。真实 LLM 模式可进一步提升到 95%+。

### Q：如何保证数据安全？
A：SQLite 数据库存储在本地，不上传云端。Docker 使用命名卷持久化数据。简历文件存储在独立的 uploads 目录。
